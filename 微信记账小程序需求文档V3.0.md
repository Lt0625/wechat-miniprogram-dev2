## 一、产品概述

### 1.1 产品定位

一款轻量、简洁、易用的微信记账工具，主打本地存储（微信小程序离线存储）、操作便捷、数据清晰，无需登录、无后端依赖，帮助用户快速记录收支、管理资产、分析消费习惯。

### 1.2 核心价值

- **便捷记账**：3步完成收支记录，操作简单，降低用户记账门槛
- **数据清晰**：直观展示收支明细、资产情况、消费统计
- **本地安全**：数据存储在微信小程序本地，无需担心隐私泄露
- **轻量无负担**：无冗余功能、无广告，运行流畅
- **智能预算**：月度预算管理，实时监控支出进度

### 1.3 开发边界

- **平台**：微信小程序
- **数据存储**：纯本地存储，使用wx.setStorageSync/wx.getStorageSync，无后端、无云同步
- **UI风格**：暖黄色主题，卡片式布局，简约浅色系
- **主题色**：主色调 #FFB347（暖橙色）、#FF9800（深橙色）、背景色 #FFF8E7（米黄色）
- **配色方案**：支出 #FF6B6B（珊瑚红）、收入 #4ECDC4（薄荷绿）、结余 #6366f1（靛蓝色）

## 五、功能规格说明

### 5.1 预算管理功能（V3.0新增）

#### 功能概述
帮助用户设定月度预算上限，实时监控支出进度，避免超支。

#### 功能流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│ 设置预算 │ → │ 记录消费 │ → │ 查看进度 │
└─────────┘    └─────────┘    └─────────┘
     ↓                              ↓
  输入金额                     进度条更新
  存储本地                     计算剩余
                               超支警告
```

#### 业务规则

1. **预算范围**：1元 ~ 9,999,999元
2. **统计周期**：自然月（1号~月底）
3. **计算公式**：
   ```
   已支出 = 当月所有 type=expense 的账单 amount 之和
   剩余 = 月度预算 - 已支出
   进度百分比 = min(已支出 / 月度预算 × 100, 100)
   ```
4. **超支判断**：剩余 < 0 时显示红色警告
5. **数据持久化**：存储在 `budgetSetting` 键中

#### 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 未设置预算 | 显示"💰 点击设置月度预算"提示 |

### 5.2 卡片式账单布局（V3.0重构）

#### 设计原则

1. **视觉层次清晰**

   - 第二层：金额（醒目颜色）
   - 第三层：时间/备注/账户（辅助信息）

   - 单屏显示 5-8 条账单
   - 重要信息一眼可见
   - 次要信息不喧宾夺主

3. **交互反馈明确**
   - 点击：缩放到98%（视觉反馈）
   - 长按：弹出操作菜单（编辑/删除）
